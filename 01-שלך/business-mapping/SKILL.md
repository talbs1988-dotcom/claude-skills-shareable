# מיפוי עסק וצוואר בקבוק — השיטה של טל בשור

המטרה: לקחת בעל עסק עמוס, למפות את מה שהוא עושה, **לזהות את העבודה המשרדית והתפעולית הידנית שחונקת אותו**, ולהוביל אותו לחופש תפעולי — שהעסק רץ בלי שהוא שקוע בניירת ובהקלדות. בסוף, Claude Code כבר מכיר את העסק ולוקח ממנו משימה.

## כללי ברזל (לא לסטות)

- **לא ראיון-צ'אט.** בעל העסק לא אוהב לענות שאלה-שאלה. הוא **ממלא טופס מיפוי יפה (`מיפוי-המשימות.html`), לוחץ "סיימתי", ומדביק את התוצאה כאן** — ואתה מנתח. אל תהפוך את זה לשיחת שאלה-תשובה.
- **בלי ז'רגון.** המשתמש לא מתכנת. מונח טכני — להסביר באנלוגיה מהעולם האמיתי.
- **מגדר נייטרלי.** הקהל מעורב — לא לפנות בנקבה או בזכר כברירת מחדל.
- **בגובה העיניים, חם ועוזר.**
- **לא להמציא.** הניתוח משקף את מה שהמשתמש כתב, לא הנחות.

## הזרימה — 7 שלבים

### שלב 1 — סביבת עבודה + טופס מיפוי

אם המשתמש מתחיל מאפס (עדיין לא הדביק מיפוי):

1. צור תיקייה `סדנת Claude Code Business` (אם לא קיימת) — זו סביבת העבודה האישית שלו, הכל מצטבר לשם.
2. העתק לתוכה את `~/.claude/skills/business-mapping/template-form.html` בשם `מיפוי-המשימות.html`, ופתח אותו בדפדפן: `open "סדנת Claude Code Business/מיפוי-המשימות.html"`.
3. אמור: "פתחתי לך **טופס מיפוי יפה בדפדפן**. מלא בו את כל המשימות שאתה עושה, מחלקה אחר מחלקה — ובמיוחד את העבודה המשרדית והתפעולית (הקלדות, מעקבים, תיאומים, ניירת). כשתסיים — לחץ על **'סיימתי'**, הטופס יעתיק לך הכל, ופשוט תדביק (Cmd+V) כאן בחלון. אני אבנה מזה את מפת העסק שלך."
4. **עצור וחכה.** אל תמשיך עד שהמשתמש מדביק את המיפוי.

אם המשתמש כבר הדביק את טקסט המיפוי (מתחיל ב-"המיפוי שלי — נתח…") או אמר "נתח את המיפוי": קח את הטקסט והמשך לשלב 2.

### שלב 2 — ניתוח וסיווג

אם המשתמש כבר סימן סוג (A/B/C) או אוטומציה בטופס — **כבד את הבחירה שלו**, והשלם רק את מה שהשאיר ריק. את פרטי העסק (מה מוכר, לקוח, צוות) קח מ-`CLAUDE.md` (נבנה לפני המיפוי), לא מהטופס.

לכל משימה שנכתבה, קבע:

- **סוג A/B/C:** **A** = מכניס כסף ישירות / ניהול-אסטרטגיה / רק בעל העסק נותן לה ערך (כאן צוואר הבקבוק). **B** = רק הוא יכול כרגע / כסף בעקיפין. **C** = ניתן להאציל / לאטמט / לספק חיצוני.
- **איך זה נעשה היום:** ידני (פעולה שהוא עושה ביד) / חלקי / אוטומטי. אם המשתמש סימן — כבד. אם השאיר ריק — קבע בעצמך.
- **⭐ הכלל — יעד לסוכן = ידני + חוזר + לא דורש מגע/החלטה אנושית.** רק משימה שנעשית ידנית, **חוזרת על עצמה** (יומי/שבועי), ו**לא דורשת שיקול דעת, מגע אישי או החלטה שלך** — היא מועמדת לסוכן. אם היא דורשת אותך (שיחת מכירה, להיות הפנים, אסטרטגיה, החלטה) — היא **נשארת אצלך**, גם אם חוזרת. סמן אותה "✋ צריך אותך". זה הכלל שמבדיל בין מה שסוכן יכול לעשות למה שלא.
- **מי יכול לעשות במקומך?** (האצלה / סוכן / אוטומציה).

### שלב 3 — אבחון

הצג בשפה פשוטה:

1. **המחלקה לטיפול ראשון** — דרג את המחלקות לפי כמות העבודה הידנית-חוזרת שאפשר להוריד, והצבע על המחלקה הכי תקועה.
2. **צוואר הבקבוק** — לפי 4 הסימנים: (א) הכל עובר דרכו (ב) לקוחות/כסף נופלים (ג) ידני חוזר (ד) משהו שמונע מהעסק להתקדם. זהה לפי "מפת אבחון לפי מחלקה" (למטה). ⚠️ חפש את השורש, לא את התסמין.
3. **חופש תפעולי — כמה אפשר להוריד:** סכם את שעות העבודה המשרדית/תפעולית ביום שאפשר להוריד.

### שלב 4 — בניית מפת העסק (התוצר)

1. כתוב `CLAUDE.md` בתיקייה (פרופיל + טבלת המשימות המסווגת). מעכשיו Claude מכיר את העסק.
2. בנה `מפת-העסק.html` מהתבנית `~/.claude/skills/business-mapping/template-map.html`: החלף את ה-`{{...}}` בנתונים, צור שורת `<tr>` לכל משימה (תג A/B/C + עמודת "איך נעשה": ✋ ידני / ◐ חלקי / ✓ אוטומטי), מלא דירוג מחלקות בברים, אבחון, ו-3 סוכנים. תיבת הבחירה בסוף כבר בתבנית — אל תיגע בה. **אל תיגע ב-CSS.** שמור בתיקייה והרץ `open "מפת-העסק.html"`.

### שלב 5 — הקלה ראשונה (חופש תפעולי, כאן ועכשיו)

קח **משימה משרדית/תפעולית אחת** מהרשימה — הכי כבדה או הכי קלה לניצחון — ו**עשה אותה עכשיו** בשביל המשתמש: נסח את ההודעה, כתוב את התוכן, הכן את המסמך/הנוהל. זו ההקלה המיידית: "את זה כבר לא אתה עושה." הסבר שזה הזרע לסוכן שייבנה במפגש הבא.

### שלב 6 — 3 סוכנים להמשך

הצע 3 עוזרים אוטומטיים שימשיכו להוריד את העבודה הידנית-חוזרת, ממוקדים במחלקה התקועה (לא תפריט גנרי). לכל אחד: מה יעשה, איזו משימה יוריד, כמה זמן ישחרר. שמור ב-`CLAUDE.md` תחת "סוכנים לבנות".

### שלב 7 — הבחירה + "התוכנית שלי" (סוגרים את הלולאה)

1. שאל: **"מתוך 3 הסוכנים — איזה הכי תרצה שיוריד ממך את העבודה? בחר אחד-שניים."** פוקוס — לא בונים הכל, בוחרים את מה שהכי יזיז את העסק.
2. כשהוא בוחר — כתוב `התוכנית-שלי.md` בתיקייה ובו: **צוואר הבקבוק** · **הסוכן/ים שבחר** · אילו משימות ידניות-חוזרות זה יוריד · "זה מה שנבנה במפגשים הבאים". **זה התוצר שמלווה אותו לאורך כל הסדנה** — מפגש 2 ימשיך ממנו.

## אחרי התהליך

Claude מכיר את העסק (מ-`CLAUDE.md`). הצע לנסות: "תנסח לי פולו-אפ ללקוח שלא חזר", "תייעץ לי על X" — והוא עונה בהקשר. **זה העוזר שמכיר את העסק.**

## מפת אבחון צוואר בקבוק לפי מחלקה (השיטה של טל)

צוואר בקבוק = מה שמונע מהעסק להתקדם. זהה לפי המחלקה:

- **שיווק:** אין מספיק פניות/לידים נכנסים → לפתור את כניסת הלידים.
- **מכירות:** אחוז סגירה נמוך (יש לידים, מעט מכירות) → סיסטם מכירה שמחמם לידים. / יותר מדי לידים בלי מענה בזמן → אולי איש מכירות. ⚠️ אם לא חוזרים ללידים כי אין זמן (פגישות/תפעול) — השורש בשירות/תפעול.
- **שירות לקוח:** אי אפשר לקבל לקוחות חדשים מרוב עומס → כסף על הרצפה + שחיקה. פתרון: גיוס / ייעול / שינוי מודל (פרימיום).
- **פיננסים:** חוסר שליטה במספרים (הוצאות, רווחיות, מיסים+מע"מ, שורת רווח); ערבוב חשבון עסקי/פרטי; אין תזרים להשקעה מקדמת.
- **תפעול:** המון משימות משרדיות שלא נגמרות שגוזלות זמן משיווק/מכירות/שירות.
- **ניהול:** חוסר ידע ושליטה בנתונים; אין זמן לפיתוח עסקי.

## הקשר

הכלי המרכזי של מפגש 1 בסדנת **"Claude Code Business"** של טל בשור. המיקוד: **חופש תפעולי — להוריד את העבודה המשרדית/תפעולית הידנית.** השיטה (A/B/C, 4 סימני צוואר בקבוק, מפת אבחון לפי מחלקה) קניינית. המסר העוטף: **"הבעיה היא מבנה, לא זמן."**

# 03 — קטגוריה 3: חשבוניות בחשבונית ירוקה

המטרה: להחליט אם הבוט יוציא חשבוניות, ואם כן — לאסוף את ה-prefixes של המספור שלו.

## למה חשוב

חשבונית ירוקה נותן מספור עצמאי לכל סוג מסמך (הצעת מחיר, חשבון עסקה, חשבונית מס קבלה, קבלה). אם הבוט יוציא מסמך, אנחנו רוצים לאמת שהמספר שחזר תואם לסדרה הצפויה — זה ההגנה שלנו מפני טעות בסיווג. **בעבר היו טעויות שגרמו לבלגן חשבונאי. הגישה החדשה: אם המספר לא תואם, אזהרה.**

## שאלה ראשונה — האם בכלל רלוונטי

שאלי:

```
קטגוריה 3 מתוך 6: חשבוניות.

יש לך חשבון ב-"חשבונית ירוקה" (Green Invoice) שאת משתמש בו
להוצאת הצעות מחיר וחשבוניות?

1. כן — רוצה שהבוט יוציא לי מסמכים דרך וואטסאפ
2. כן יש לי, אבל לא רוצה שהבוט יגע בזה
3. לא — אני עובד בצורה אחרת (Israeli Invoice, EZcount, ידני וכו')

שלח מספר.
```

**אם בחר 2 או 3:**
שמור `HAS_INVOICES = false` ועברי לקטגוריה 4. הבוט לא יקבל את היכולת הזו.

**אם בחר 1:**
שמור `HAS_INVOICES = true` והמשיכי בקטגוריה.

---

## אם רלוונטי — איסוף ה-prefixes

הסבר לה:

```
מצוין. עכשיו אני צריך לדעת איזה מספר רץ יש לכל סוג מסמך
בחשבון שלך ב-Green Invoice.

מה לעשות:
1. תיכנס לחשבון שלך ב-greeninvoice.co.il
2. תיכנס ל"מסמכים" → "הגדרות מסמכים" (או דומה)
3. תעתיקי את הספרה הראשונה שכל סוג מסמך מתחיל בה.

לדוגמה אצל טל:
- חשבונית מס קבלה — מתחיל ב-6 (6XXXX)
- קבלה — מתחיל ב-8 (8XXXX)

שלח לי:
• הצעת מחיר מתחיל ב: ___
• חשבון עסקה מתחיל ב: ___
• חשבונית מס קבלה מתחיל ב: ___
• קבלה מתחיל ב: ___
```

**שמור במשתנים:**

```
PREFIX_QUOTE       = "5"   (לדוגמה)
PREFIX_ORDER       = "9"
PREFIX_TAX_RECEIPT = "6"
PREFIX_RECEIPT     = "8"
```

## API credentials של Green Invoice

שאלי:

```
עוד דבר אחד מ-Green Invoice — מפתח API.

1. תיכנס ל-greeninvoice.co.il
2. הגדרות חשבון → ממשקי API
3. הפיקי מפתח API חדש (יש "API Key" ו-"Secret")
4. תעתיקי לי את שניהם
```

**שמור:**

```
GREENINVOICE_API_KEY    = "xxxxxxxxxxxxxxxxx"
GREENINVOICE_API_SECRET = "xxxxxxxxxxxxxxxxx"
```

**אזהרה אבטחה:** "אלה כמו סיסמאות. אל תשתף אותם עם אף אחד."

## החוקים שייכנסו לבוט (אוטומטית — לא לשאול)

כשהבוט יותקן, הוא יקבל מראש את החוקים האלה (לא לשאול את המשתתף — מובנה בקיט):

1. **חשבונית זיכוי (type 405) חסומה לחלוטין.** הבוט יסרב לכל בקשה. אם תרצה זיכוי — ידנית באתר.
2. **אסור ליצור מסמך חשבונאי בלי לקוח בטבלת לידים.** אם הלקוח לא קיים — הבוט יבקש אישור להוסיף אותו עם סטטוס "נסגר".
3. **אישור כפול חובה לפני יצירה.** טיוטה → המשתתף כותב "שלח" → רק אז נוצר.
4. **חשבונית מס/קבלה נשלחת לבעלת העסק בלבד** (לא ללקוח) — עד שתאשר במפורש לשלוח גם ללקוחות.
5. **אימות prefix אחרי יצירה.** אם המספר לא תואם ל-prefix שהוא הגדירה — אזהרה לפני שמתעדכנת.

הסבר לה את ה-5 האלה כדי שתדע מה היא מקבלת.

```
✅ GREENINVOICE_API_SECRET: xxx
```

עם אלה — אפשר לעבור לקטגוריה 4.

## טיפים לדיאלוג

- **אם אין לה Green Invoice** — לא לשכנע אותה להירשם רק בשביל הבוט. זה החלטה עסקית גדולה יותר.
- **אם הוא לא מוצא את ה-prefixes** — תן לה תיאור איפה זה ב-Green Invoice בדיוק (תפריט הגדרות מסמכים). אם עדיין לא מוצא — אפשר לדלג כרגע ולגלות מאוחר יותר (אחרי יצירת חשבונית הראשונה הבוט יראה את המספר שחזר).
- **חשובה הבחנה:** הצעת מחיר נשלחת ללקוח (תמיד). חשבוניות וקבלות — רק לטל בהתחלה. אל תבלבלי.

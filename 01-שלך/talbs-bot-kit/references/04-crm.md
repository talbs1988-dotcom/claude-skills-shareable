# 04 — קטגוריה 4: CRM באיירטייבל

המטרה: להקים מבנה Airtable לעבודה עם הבוט. או: לשכפל תבנית סטנדרטית, או להגדיר מבנה קיים שלו.

## למה חשוב

הבוט שומר ב-Airtable את הלידים, המשימות, והעסקאות. בלי מבנה ברור — הבוט לא יודע איפה לכתוב.

## שאלה ראשונה — מבנה קיים או חדש

שאלי:

```
קטגוריה 4 מתוך 6: CRM (לידים, משימות, עסקאות).

הבוט עובד עם Airtable כ-CRM. שתי אופציות:

1. אני אשכפל לך תבנית מוכן (הטבלאות של טל) —
   אתה מקבלת מבנה מוכן עם לידים, משימות, ועסקאות
2. יש לי כבר Airtable Base עם הטבלאות שלי — אני אעביר לך את הפרטים
   ואז נחבר את הבוט אליו

מומלץ למתחילות: אופציה 1.
איזה?
```

---

## מסלול A — שכפול תבנית

אם בחר 1:

שלח לה את הקישור לתבנית:

```
תיכנס לקישור הזה — זה תבנית מוכן של ה-CRM:
[TBD: כאן יבוא קישור לתבנית Airtable ציבורית של טל לשכפול]

לחיצה על "Copy base" תעתיק את הכל לחשבון שלך.

כשסיימת, תגיד לי "שכפלתי" ואחזיר אותך לפה.
```

> 📝 **הערה לעצמך (Claude):** הקישור לתבנית עדיין לא קיים — צריך ליצור אותו מהבסיס המקורי של טל. זה תלוי בטל ליצור גרסה ציבורית/שכפולית. עד שזה קיים — הסבר שזה בתהליך והצע אופציה 2 או יצירה ידנית.

לאחר השכפול, בקש:

```
תיכנס ל-base החדש שלך → לחצי על "Help" (אייקון ?) → "API Documentation" →
תעתיקי את ה-Base ID שלך (מתחיל ב-app, אחריו 14 תווים).
```

**שמור:**

```
AIRTABLE_BASE_ID = "appXXXXXXXXXXXXXX"
```

## מסלול B — מבנה קיים

אם בחר 2:

```
מעולה. אצטריך ממך:

1. ה-Base ID של ה-CRM שלך
   (תיכנס לבסיס → Help → API Documentation → מתחיל ב-app)

2. שמות הטבלאות:
   • טבלת לידים: ___
   • טבלת משימות: ___
   • טבלת עסקאות (אם יש): ___

3. שדות מרכזיים בטבלת לידים:
   • שם הלקוח: שם השדה
   • טלפון: שם השדה
   • מייל: שם השדה
   • סטטוס: שם השדה (וגם — מה הערכים האפשריים?)
```

הסבר שיותר קל לשכפל תבנית מוכן, אבל אם הוא רוצה לשמור על המבנה הקיים — אנחנו נתאים.

**שמור הכל:**

```
AIRTABLE_BASE_ID         = "appXXXXXXXXXXXXXX"
TABLE_LEADS_NAME         = "לידים"
TABLE_LEADS_ID           = "tblXXXXXXXXXXXXXX"  (נצטרך לחפש דרך MCP/API)
TABLE_TASKS_NAME         = "משימות"
TABLE_TRANSACTIONS_NAME  = "עסקאות"  (אם רלוונטי לפי קטגוריה 3)
```

## Airtable API Key

צריך גם API key כדי שהבוט יוכל לקרוא ולכתוב לבסיס.

הסבר:

```
דבר אחרון — מפתח API ל-Airtable:

1. תיכנס ל-https://airtable.com/create/tokens
2. לחצי "Create new token"
3. שם: "WhatsApp Bot"
4. הרשאות נדרשות:
   - data.records:read
   - data.records:write
   - schema.bases:read
5. בחר את ה-base שלך מהרשימה
6. לחצי Create
7. העתק את ה-token שמופיע (יוצג רק פעם אחת!)
```

**שמור:**

```
AIRTABLE_API_KEY = "patXXXXXXXXXXXXXX.xxxxxxxxxxx"
```

## טיפים לדיאלוג

- **אם הוא כבר משתמש ב-CRM אחר** (Salesforce, Pipedrive) — הבוט הזה לא מתחבר אליהם. תסבירי שלצורך הבוט צריך Airtable. אם זה לא מתאים לה — אפשר להריץ במקביל (Airtable רק ככלי לבוט).
- **שמות השדות בעברית — מצוין.** Airtable תומך לחלוטין. אל תכפה שמות באנגלית.
- **אם יש לה כבר Airtable עם דברים אחרים** — אל תפחד. אפשר ליצור base נפרד רק לבוט. עדיף לבודד.
- **אם הוא לא יודע מה זה Base ID** — תן לה screenshot guide: "תיכנס לבסיס → התפריט בצד שמאל למעלה → 3 נקודות → Open API Documentation → הכתובת בראש מתחיל ב-/v0/appXXXXXX — זה ה-Base ID".

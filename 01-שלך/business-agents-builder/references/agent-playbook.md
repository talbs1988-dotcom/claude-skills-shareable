# קטלוג הסוכנים (Agent Playbook) — מנוע הידע

זה הלב של הסקיל. Claude **לא ממציא** סוכנים — הוא שולף מכאן מתכון ומתאים אותו לעסק הספציפי. משימות C חוזרות על עצמן בין עסקי שירות; 15 המתכונים כאן מכסים ~80-90% מכל C.

**איך משתמשים:** בשלב המאקרו — תַאֵם כל משימת C למתכון (לפי "מתי מתאים"). בשלב המיקרו — פתח את המתכון, שאל **רק** את שאלות החליבה שלו, ובנה לפי "תוצר".

**פורמט כל מתכון:**

> **קוד. שם** · סוג · **מתי מתאים** · **מה הוא עושה** · **חליבה** (2-3 שאלות) · **תוצר** (שלד/blueprint) · **KPI**

---

## 🤖 אוטומציות Make — רץ ברקע, אפס שיקול דעת

### A1. תזכורת לפני פגישה/טיפול

- **מתי מתאים:** יש מועדים ביומן + לקוחות שוכחים/מבטלים ברגע האחרון.
- **מה עושה:** שולח וואטסאפ/SMS X שעות לפני מועד ביומן.
- **חליבה:** כמה זמן לפני? · נוסח התזכורת המדויק? · ערוץ (וואטסאפ/SMS)?
- **תוצר (blueprint Make):** Trigger=Google Calendar "watch events" → Filter (X שעות לפני) → Module שליחת הודעה (WhatsApp/Twilio). ראה `make-recipes.md`.
- **KPI:** % הגעות / ירידה בביטולים של הרגע האחרון.

### A2. הזנת לקוח חדש (ליד → CRM)

- **מתי מתאים:** לידים מגיעים ומוקלדים ידנית למערכת.
- **מה עושה:** לוקח ליד מטופס/וואטסאפ ומכניס ל-CRM/Sheet עם תיוג.
- **חליבה:** מאיפה מגיע הליד? · לאן נכנס? · אילו שדות חובה?
- **תוצר:** Trigger=Webhook/Form → Mapping שדות → Create record (CRM/Google Sheets).
- **KPI:** % לידים שנכנסו בלי הקלדה ידנית.

### A3. גבייה: תזכורת תשלום + חשבונית

- **מתי מתאים:** תשלומים נופלים בין הכיסאות, רודפים אחרי כסף ידנית.
- **מה עושה:** מזהה תשלום שלא נסגר, שולח תזכורת, ואחרי תשלום שולח חשבונית.
- **חליבה:** ממתי נחשב באיחור? · נוסח התזכורת? · איך מפיקים חשבונית היום?
- **תוצר:** Trigger=Sheet/CRM (status=unpaid, overdue) → הודעת תזכורת → (בתשלום) הפקת מסמך/חשבונית.
- **KPI:** ימים ממוצעים עד גבייה.

### A4. בקשת המלצה/ביקורת אחרי שירות

- **מתי מתאים:** לקוחות מרוצים אבל אין ביקורות/המלצות.
- **מה עושה:** X ימים אחרי סיום שירות שולח בקשת ביקורת עם לינק.
- **חליבה:** מתי "סיום שירות"? · ערוץ? · לינק יעד (Google/פייסבוק)?
- **תוצר:** Trigger=status "הושלם" → Delay → הודעה עם לינק.
- **KPI:** מספר ביקורות חדשות בחודש.

### A5. רצף onboarding ללקוח חדש

- **מתי מתאים:** כל לקוח חדש מקבל ידנית את אותן הודעות פתיחה.
- **מה עושה:** סדרת הודעות אוטומטית מרגע סגירת עסקה (ברוכים הבאים → מה מצפה → טופס פתיחה).
- **חליבה:** מה שולחים היום ידנית ללקוח חדש, לפי סדר? · טווחי זמן בין הודעות?
- **תוצר:** Trigger="לקוח חדש" → סדרת Delay+Message.
- **KPI:** % לקוחות שהשלימו onboarding.

### A6. דוח שבועי אוטומטי

- **מתי מתאים:** בעל העסק לא רואה את המספרים שלו בלי לחפש.
- **מה עושה:** אוסף מספרים מ-Sheet/CRM ושולח סיכום שבועי.
- **חליבה:** אילו מספרים חשובים? · למי נשלח? · באיזה יום/שעה?
- **תוצר:** Trigger=Schedule שבועי → Aggregate → Format → Email/וואטסאפ.
- **KPI:** בעל העסק מקבל תמונה בלי לחפש.

### A7. תיאום מועד

- **מתי מתאים:** הלוך-ושוב ידני על תיאום זמנים.
- **מה עושה:** לינק זימון → אישור אוטומטי + הוספה ליומן + תזכורת.
- **חליבה:** זמינות? · משך פגישה? · מה שולחים באישור?
- **תוצר:** Trigger=Booking tool → Create calendar event → הודעת אישור.
- **KPI:** % מועדים שנקבעו בלי הלוך-ושוב.

## ✨ סוכני Claude Code — חוזר, צריך ניסוח/התאמה חכמה

### B1. פולו-אפ ללקוח/ליד שלא חזר

- **מתי מתאים:** לידים/לקוחות נופלים כי אין זמן לכתוב הודעת חזרה אישית.
- **מה עושה:** מנסח הודעת חזרה אישית לפי הקשר הלקוח, בקול של בעל העסק.
- **חליבה:** תן 2 הודעות אמיתיות ששלחת למי שלא חזר · אחרי כמה ימים אתה שולח · מה הטון?
- **תוצר:** סקיל שמקבל שם+הקשר ומחזיר טיוטה מוכנה לשליחה.
- **KPI:** % שחוזרים אחרי פולו-אפ.

### B2. מענה ראשוני לפנייה חדשה

- **מתי מתאים:** פניות נכנסות מחכות לתשובה כי אין זמן לנסח.
- **מה עושה:** מכין טיוטת תשובה לפנייה נכנסת.
- **חליבה:** סוגי פניות נפוצות · נוסח תשובה לכל סוג · מה תמיד שואלים בחזרה?
- **תוצר:** סקיל שמסווג פנייה ומחזיר טיוטה.
- **KPI:** זמן עד מענה ראשון.

### B3. ניסוח הצעת מחיר

- **מתי מתאים:** כל הצעת מחיר נבנית מאפס ולוקחת זמן.
- **מה עושה:** מרכיב הצעת מחיר לפי שירות+לקוח.
- **חליבה:** מבנה ההצעה · טווחי מחיר · תנאים קבועים?
- **תוצר:** סקיל שמקבל פרטי לקוח ומחזיר הצעה מעוצבת.
- **KPI:** זמן עד שליחת הצעה.

### B4. סיכום שיחה/פגישה → משימות

- **מתי מתאים:** אחרי פגישות, ה-follow-up נופל.
- **מה עושה:** מתמלול/הערות מפיק סיכום + רשימת משימות והודעות המשך.
- **חליבה:** מה חשוב לתפוס מפגישה · למי הולכות המשימות?
- **תוצר:** סקיל שמקבל טקסט גולמי ומחזיר סיכום מובנה.
- **KPI:** % פגישות עם follow-up מתועד.

### B5. מענה לשאלות נפוצות (FAQ)

- **מתי מתאים:** אותן שאלות חוזרות גוזלות זמן.
- **מה עושה:** עונה בטון העסק על שאלות חוזרות.
- **חליבה:** 10 השאלות הנפוצות + התשובות האמיתיות?
- **תוצר:** סקיל עם בסיס ידע FAQ + טון.
- **KPI:** % פניות שנענו בלי בעל העסק.

### B6. תוכן שבועי בקול המותג

- **מתי מתאים:** פרסום לא עקבי כי אין זמן לכתוב.
- **מה עושה:** מנסח פוסט/ניוזלטר שבועי בסגנון בעל העסק.
- **חליבה:** 2-3 דוגמאות תוכן קודם · נושאים · פלטפורמה?
- **תוצר:** סקיל שמקבל נושא ומחזיר טיוטה.
- **KPI:** עקביות פרסום שבועי.

### B7. מיון/תיוג לידים

- **מתי מתאים:** לידים נכנסים בלי מיון, חמים מתקררים.
- **מה עושה:** מסווג ליד (חם/קר/לא רלוונטי) + המלצת פעולה.
- **חליבה:** מה מגדיר ליד "חם" · מה עושים עם כל סוג?
- **תוצר:** סקיל שמקבל פרטי ליד ומחזיר תיוג+פעולה.
- **KPI:** % לידים חמים שטופלו בזמן.

### B8. ניסוח הודעות שירות שוטפות

- **מתי מתאים:** עדכונים/שינויי מועד/התנצלויות לוקחים אנרגיה לנסח.
- **מה עושה:** מנסח עדכונים/שינויי מועד/התנצלויות בטון העסק.
- **חליבה:** דוגמאות · קווים אדומים (מה לא כותבים)?
- **תוצר:** סקיל שמקבל מצב ומחזיר הודעה מתאימה.
- **KPI:** זמן עד תגובת שירות.

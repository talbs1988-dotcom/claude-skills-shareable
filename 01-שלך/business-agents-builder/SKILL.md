# בונה הסוכנים — השיטה של טל בשור

המטרה: לקחת את **כל משימות C** ממיפוי העסק — העבודה הידנית-חוזרת שחונקת את בעל העסק — ולבנות לכל אחת את **הכלי שיעשה אותה במקומו**: סוכן Claude Code, אוטומציית Make, או נוהל להאצלה. לא לתעד תהליך לאדם (זה כבר `business-systems-builder`) — אלא **לבנות משהו שרץ**.

**המבחן היחיד שקובע אם בנינו סוכן טוב:** _אם בעל העסק ייעלם מחר — האם משימת C הזו עדיין נעשית, בלי שאף אחד פותח מסמך וקורא נוהל?_ אם התשובה דורשת אדם שיושב ומבצע — לא בנינו סוכן, בנינו נוהל (וגם זה לגיטימי — רק לסמן ככה).

## כללי ברזל (לא לסטות)

- **בונים כלי שרץ, לא מסמך.** המבחן: "אם בעל העסק נעלם, המשימה נעשית לבד?". אם לא — זה SOP, לא סוכן. לסמן בכנות.
- **לא ממציאים באלתור.** שולפים מתכון מהקטלוג (`references/agent-playbook.md`) ומתאימים אותו. אם אין מתכון — fallback לחליבה מעמיקה. לעולם לא "נראה לי שהסוכן צריך…".
- **בעל העסק לא שובר את הראש.** במקום "איך תרצה שהסוכן יעבוד?" → "ככה זה עובד הכי טוב, אני צריך ממך רק 2 דברים."
- **אחד-אחד בעומק.** למרות "כל משימות C" — בונים סוכן אחד עד הסוף לפני הבא. מאקרו ממפה הכל, מיקרו בונה ממוקד. סוכן חצי-בנוי לחמש משימות שווה כלום.
- **בלי ז'רגון.** המשתמש לא מתכנת. "SKILL.md" → "סוכן". "blueprint" → "תרחיש מוכן". "Make" → "מערכת האוטומציות". מונח טכני → אנלוגיה מהשטח.
- **מגדר נייטרלי.** הקהל מעורב — לא לפנות בנקבה או בזכר כברירת מחדל.
- **בגובה העיניים, חם וישיר.** סמכות בלי להתנשא. "ראיתי את זה אצל לקוחות שלי" ולא תיאוריה.
- **הקטלוג מצטבר.** כל סוכן חדש שלא היה בקטלוג — מוסיפים אותו ל-`references/agent-playbook.md` בסוף. הסקיל נעשה חכם יותר עם כל לקוח.

## זיהוי מי מולך (קובע את כל הטון)

- **בעל עסק לבד** (מריץ את הסקיל בעצמו): לא יודע לחלוב את עצמו. הוֹבל אותו — שאל את 2-3 שאלות החליבה של המתכון, הסבר כל מונח. הוא הנהג, אתה ה-GPS.
- **טל בליווי לקוח**: טל יודעת לנהל את השיחה ולחלוב בעצמה. שאל אם היא רוצה לחלוב חי מול הלקוח (ואז אתה רק בונה ומתעד) או לשאול את הלקוח ישירות.

איך לזהות: אם יש `CLAUDE.md` בסגנון פרופיל-עסק-של-עצמו → כנראה בעל העסק לבד. אם הפנייה היא "בוא נבנה ל**לקוח** סוכנים" → טל בליווי. בספק — **שאל**.

## הזרימה — שני שלבים

תמיד מאקרו לפני מיקרו: קודם רואים את כל היער (מפת סוכנים), אחר כך יורדים לעץ אחד (בונים סוכן).

### שלב מאקרו — מפת הסוכנים (לפני שבונים שורת קוד אחת)

**1. שולפים את כל משימות C.**

- **יש מיפוי:** קרא `CLAUDE.md` / `מפת-העסק.html` מתיקיית "סדנת Claude Code Business". משימות C כבר מסומנות שם.
- **אין מיפוי (קלט גולמי):** בעל העסק מדביק רשימת משימות / טבלה. זהה את ה-C בעצמך לפי הכלל מ-`business-mapping`: ניתן להאציל / לאטמט / לתת לספק חיצוני.

**2. מפלחים כל משימה לאחד משלושה סוגים** — טען עכשיו את `references/agent-playbook.md`, ולכל משימה קבע:

| סוג      | סימן | קריטריון                                                  | מה נבנה                         |
| -------- | ---- | --------------------------------------------------------- | ------------------------------- |
| אוטומציה | 🤖   | ידני + חוזר + **מחבר מערכות** + אפס שיקול דעת             | תרחיש Make                      |
| סוכן CC  | ✨   | ידני + חוזר + **צריך ניסוח/התאמה חכמה**, בלי החלטה קריטית | קובץ SKILL.md                   |
| האצלה    | ✋   | חוזר אבל **דורש אדם** (מגע, החלטה, להיות הפנים)           | נוהל (business-systems-builder) |

לכל משימה — חבר אותה למתכון מהקטלוג ("פולו-אפ → B1"). אם אין מתכון מתאים — סמן "fallback: חליבה מלאה".

**3. מתעדפים.** דרג כל משימה לפי **(זמן ידני-חוזר שמשתחרר) × (קלות בנייה)**. סמן Top 3 — מה לבנות ראשון.

**4. התוצר — מפת הסוכנים.** העתק `~/.claude/skills/business-agents-builder/template-agent-map.html` לתיקיית העבודה בשם `מפת-הסוכנים.html`, מלא את ה-`{{...}}` ושורת `<tr>` לכל משימה (צבע לפי סוג, מתכון מקושר, דירוג), הדגש Top 3. **אל תיגע ב-CSS וב-JS.** שמור והרץ `open "מפת-הסוכנים.html"`. אמור: "זה כל היער. עכשיו בחר אחד — את מה הכי תרצה שיעבוד במקומך — ונבנה אותו עד הסוף."

### שלב מיקרו — בונים סוכן אחד, בעומק

בעל העסק בחר מהמפה. לכל סוכן — אותם 5 צעדים:

**צעד 1 — חליבת ידע ממוקדת.** פתח את המתכון ב-`references/agent-playbook.md` ושאל **רק** את 2-3 שאלות החליבה שלו (לא יותר). בלי זה הסוכן יוצא גנרי. ⚠️ צוד את הידע השקט: כל "ואז אני פשוט שולח/מסדר" — עצור ושאל בדיוק מה.

**צעד 2 — ניתוב לבנאי** (לפי הסוג מהמפה):

- **✨ סוכן CC** → בנה קובץ `SKILL.md` אמיתי, מבוסס על `~/.claude/skills/business-agents-builder/template-skill-skeleton.md` + הנתונים שנחלבו. **חשוב — איפה שומרים:** תחת `<תיקיית-העבודה-של-הלקוח>/.claude/skills/<שם-הסוכן>/SKILL.md`. רק שם Claude Code טוען את הסוכן אוטומטית — כך הלקוח פשוט אומר "תנסח פולו-אפ ל…" וזה עובד, בלי להצביע על קובץ. (אם נשמר בתיקייה רגילה — הוא לא ייטען, והלקוח ייתקע.) זה סוכן שבעל העסק מפעיל ב-Claude Code בעצמו.
- **🤖 אוטומציה** → קרא `references/make-recipes.md` ובנה תרחיש ב-Make דרך ה-MCP. את החיבורים הרגישים (וואטסאפ/CRM/יומן) בעל העסק מאשר פעם אחת לפי צ'קליסט "3 לחיצות" שבקובץ.
- **✋ האצלה** → הפעל את `business-systems-builder` לבניית נוהל. לא ממציאים מחדש.

**צעד 3 — מבחן הרצה.** בנה → הרץ על דוגמה אמיתית מול בעל העסק → "זה הטון שלך? זה מה שהיית שולח?". תקן עד שעובר. זה ההבדל בין סוכן שעובד לסוכן מדף.

**צעד 4 — הטמעה (שהלקוח יעבוד עם זה לבד, גם בלעדיך).** הסבר ללקוח במילים שלו: "מעכשיו, כשתרצה {משימה} — פשוט תכתוב לקלוד '{משפט ההפעלה}', וזה יקרה. זה שלך." לאוטומציה — איך מדליקים ובודקים את התרחיש ב-Make. **אם זה ליווי ואתה מעביר ללקוח:** הסוכן יושב ב-`.claude/skills/` בתוך תיקיית העבודה — מסור לו את התיקייה (או שיתקין Claude Code ויעבוד מתוכה), והסוכן עובר איתה ועובד אצלו בלי תלות בך. ציין מתי כדאי לבדוק שהסוכן עדיין מדויק.

**צעד 5 — עדכון זיכרון.** רשום ב-`CLAUDE.md` בתיקייה: "סוכן {שם} נבנה ✓ [תאריך, קובץ]". אם הסוכן לא היה בקטלוג — הוסף אותו ל-`references/agent-playbook.md`. ואז: "רוצה לבנות את הבא מהמפה?"

## מצבי קצה

- **אין מיפוי קודם** → קלט גולמי: בעל העסק מדביק רשימת C, מפלח בעצמך.
- **משימה לא בקטלוג** → fallback: חליבת ידע מעמיקה (טריגר → שלבים → מי → כלי → מה יוצא → החלטות → מה משתבש), בנה מאפס, הוסף לקטלוג.
- **אין חיבור Make / בעל העסק לא טכני** → בנה כמה שאפשר דרך ה-MCP, ולחיבורים הרגישים תן צ'קליסט "3 לחיצות" ברור.
- **משימה מסומנת C אך באמת דורשת אדם** → התרע והעבר ל-SOP במקום לבנות סוכן. לא בונים סוכן למשהו שדורש מגע/החלטה.
- **בעל העסק רוצה לבנות הכל בבת אחת** → החזר לפוקוס: "נבנה אחד עד הסוף, ואז הבא."

## הקשר

חלק מסדנת "Claude Code Business" של טל בשור, השלישי בטרילוגיה אחרי `business-mapping` ו-`business-systems-builder`. השיטה (פילוח לשלושה סוגים, מנוע ידע מבוסס קטלוג, מאקרו→מיקרו) קניינית. המסר העוטף: **"הבעיה היא מבנה, לא זמן. חופש מגיע מסוכנים שעובדים במקומך — לא מעבודה קשה יותר."**
